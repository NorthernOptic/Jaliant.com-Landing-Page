The aim is to build:

The most easy-to-use privacy conscious distributed social network; bringing encrypted communication to all. Completely decentralised.

Recognising that it is vital that privacy software must be accessible to the widest audience, it is our aim to marry privacy, simplicity and security; building the first encrypted, open-source social network accessible to everyone.

We intend to use bank grade encryption, distribute the network across the internet to overcome censorship and make the source code freely accessible via GitHub.

It makes censorship and surveillance harder:

The network is decentralised, there is no centralised data store for anyone to spy on. Messages are encrypted making data surveillance harder compared to networks that leave them in plain text. Moreover this is easier to use, no complicated configuration files to write or programs to install; it's as easy as logging into a website.

This is privacy by design, meaning you don't have to trust us to keep your data secure; you can trust mathematics.

As anyone can set-up a node, there isn't just one place to censor; this makes censorship far more difficult.

To sum it up...

Jaliant seeks to be a distributed network boasting powerful encryption techniques but being easy to use with a focus on usability, allowing everyone to protect their privacy with encryption.

For the end user it's as easy as them signing up on a website with just a username and password, encryption keys are generated without the user even noticing. The user can then post to the network, share their links and media, follow pages and add friends. They don't need to know anything anymore complicated as everything else is done behind the scenes (though the geeky data is available for coding wizards).

Jaliant doesn't just help secure your privacy; if one node is blocked by a censored internet connection, simply connect to a different one which is unblocked and you can access another users profile. It helps make the flow of data a lot more free.

You can host your own data by running your own node; those who can't will be able to use our first node (accessible at jal.io) or alternatively use other open nodes which they can find through our Node Directory. We also are keen to integrate a Tor Hidden Service with our initial node if we can, allowing better anonymity for those who want to remain completely anonymous while using the service.
